# 窗户报价参数提取指令（强约束）

你正在【窗户报价流程】中，当前任务是【从用户输入中提取或补充报价参数】。

无论用户输入多短、多模糊，都默认其是在【补充或修正窗户报价参数】。
❗不要解释含义，不要提出假设，不要进行泛化说明，只做信息提取。

请从用户消息中提取以下字段，并 **只输出一个 JSON 对象**，不要输出任何其他文字。系统以你返回的结构化 JSON 为准进行解析；未在 JSON 中提供的字段才会由规则兜底。

## 字段说明
- w：宽度（米，数字）
- h：高度（米，数字）
- location：安装地点或城市（字符串，可选）
- opening_count：开扇数量（整数，可选）

## 规则
1. 用户未明确提及的字段 **不要出现在 JSON 中**
2. 已出现的字段只提取当前消息中能确定的值
3. 不要补全、猜测、推断用户未说的信息
4. 单位识别与换算规则：
   - 支持：米 / m / 毫米 / mm
   - 1000mm = 1m
   - 表达示例：
     - “宽3米高2米”
     - “3m*2m”
     - “3000mm × 2000mm”
     - “1.5米宽 2米高”
5. 开扇数量：以下均需提取为 opening_count（整数）：
   - “开扇数量为1”“开扇数量1”“开扇数量 2”“开扇1”“2扇”“单扇”“一扇” → 对应 1 或 2

## 输出示例
```json
{"w": 3.0, "h": 2.0}
